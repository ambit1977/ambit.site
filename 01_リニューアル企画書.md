## 1. 背景と現状課題

### 1.1 現状サイトの整理

| 項目 | 現状 |
| --- | --- |
| 構成 | Home / About Us / Services / Contact の4ページ構成 |
| トップ訴求 | 「オーダーメイド型のWEBマーケティング支援」 |
| サービス掲載 | SEO / 広告運用 / 計測設定 / MA / DMP / メディア収益化 / 機械学習 など8領域を列挙 |
| プロフィール | 代表者の連絡先のみ |
| 実績 | 掲載なし |
| ビジュアル | 最小限（テンプレ感のあるトップ画像のみ） |

### 1.2 課題サマリ

1. **専門性が伝わらない** — 領域が広く列挙されている一方で、「何が一番強いのか」「他のフリーランス／コンサルとどう違うのか」が読み取れない。
2. **信頼の根拠が不足** — 実績・経歴・クライアント業界・案件タイプの開示がなく、初見の問い合わせハードルが高い。
3. **想起のフックがない** — ブログ・ナレッジ発信がなく、「データガバナンス」「GA4移行」「Cookieless計測」などのホットトピックでの想起が取れていない。
4. **問い合わせ動線が弱い** — 各ページに「次の一歩」を促す導線設計がなく、Contactは事務的なフォームのみ。
5. **トーンが事業主としての温度感を欠く** — 「誰が」「どんな姿勢で」支援してくれるのかが伝わらず、結果として代理店との差別化ができていない。

## 2. リニューアルの目的とゴール

### 2.1 KGI（最終ゴール）

**インバウンド問い合わせ経由の新規コンサル契約を獲得する。**

- 既存はリファラル中心と想定。サイトは「紹介された人が安心して問い合わせるための場」かつ「指名で見つけてもらうための場」の二役を担う。

### 2.2 KPI（運用指標）

| カテゴリ | 指標 | 初期目標 |
| --- | --- | --- |
| 集客 | 月間セッション数 | 【ベースライン計測後に設定】 |
| 集客 | 指名検索 + 自然検索流入比率 | 6か月で自然検索 30% 以上 |
| エンゲージ | サービス詳細ページの平均閲覧時間 | 60秒以上 |
| エンゲージ | Insights記事の読了率 | 50% 以上 |
| CV | Contactフォーム送信数 | 月3件 |
| CV | 「相談前提」CTA（カジュアル面談）クリック率 | サービス詳細PV比 8% |

### 2.3 サブ目的

- **採用文脈（業務委託・スポット相談）** にも耐える「経歴の信頼性」を提示。
- **代理店・制作会社のパートナー担当** に対し、「組める専門家」としての立ち位置を示す。

## 3. ターゲットとペルソナ

### 3.1 ペルソナA: 事業会社のマーケ責任者・担当者（主ターゲット）

- 役職: マーケ部長 / Digital Lead / CRM Manager / アナリティクス担当
- 業種: EC、SaaS、メディア、金融、人材、消費財などデータ活用が事業KPIに直結する業種
- 課題: GA4移行後の計測設計が混乱、Cookieless対応、データ基盤と広告配信の連携、社内のデータガバナンス整備
- 意思決定基準:
  - 「上流の戦略」だけでなく「実装まで踏み込める」か
  - 代理店の言いなりにならずに発注者側に立ってくれるか
  - 個人事業主としてのレスポンスの速さ・柔軟性

### 3.2 ペルソナB: 代理店・制作会社のパートナー担当

- 役職: アカウントプランナー / プロデューサー / アナリティクス部門マネージャー
- 課題: クライアントから求められる専門領域（DMP、CDP、計測実装、ML）に自社で対応しきれない
- 意思決定基準:
  - 自社案件にスポットで参加してくれる柔軟性
  - クライアント前に出しても安心な人物像・経歴
  - 守備範囲の広さと、それを束ねた語り口

### 3.3 ペルソナC: リクルーター・採用文脈

- 形態: 業務委託の長期契約 / フラクショナルCMO・CDO・Head of Data 的なポジション
- 確認したい情報: 経歴、得意領域、稼働形態、料金感、過去のロール

## 4. ポジショニングとメッセージ戦略

### 4.1 提供価値の中核

> **「事業会社の中の人」として伴走する、データガバナンス × WEBアナリティクス × アドテクの専門家。**

- フリーランスでありながら、**上流（ガバナンス・基盤設計）から下流（広告運用・MLによる打ち手）まで横断**できる。
- 代理店の窓口でも、社内の専任マネージャーでもない、**第三のポジション**を担う。

### 4.2 差別化軸（3点に絞る）

1. **ガバナンス × 実装の両利き** — 規程・ルール整備だけで終わらず、GTM/GA4/CDP/広告タグまで自分で触れる。
2. **発注者目線の独立性** — 媒体・ツールベンダーから独立した立場で意思決定を支援。
3. **個人事業主だからこその密度** — 顧問契約形態で「いつでも相談できるマーケ部門の隣の席」を提供。

### 4.3 タグライン候補

- A案: **"Marketing, with governance."**（堅め・グローバル寄せ）
- B案: **"データを、事業の意思決定言語にする。"**（和文・事業会社寄せ）
- C案: **"上流から、運用まで。"**（シンプル・実装力訴求）

→ **推奨: A案 + 和文サブコピー（B案）併用**

## 5. スコープ（リニューアル範囲）

### 5.1 含めるもの

- 全ページの構成・コンテンツ再設計（後述IA参照）
- デザイン全面刷新（Claude Designを活用してモック生成）
- お問い合わせフォームの改修（カジュアル面談導線追加）
- Insights（記事）機能の追加
- 計測基盤の刷新（GA4 + サーバーサイドGTM + 簡易CDPの自己ドッグフーディング）

### 5.2 含めないもの（フェーズ2以降）

- 多言語化（英語版は次フェーズ）
- 会員ログイン、有料コンテンツ
- 自動メール配信・MAツール接続

### 5.3 技術スタック案

| レイヤ | 候補 | 備考 |
| --- | --- | --- |
| フロント | Next.js (App Router) + TypeScript | SSG中心、ISRでInsights更新 |
| スタイル | Tailwind CSS + shadcn/ui | Claude Designの出力と相性◎ |
| CMS | MicroCMS or Contentlayer (MDX) | Insightsの運用負荷を抑える |
| ホスティング | Vercel | 既存ドメイン `ambit.go2020.tokyo` を維持 or 独自ドメイン取得 |
| 計測 | GA4 + sGTM + Plausible（補助） | 自社サイト＝ショーケース化 |
