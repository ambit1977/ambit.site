## 4. CTA設計

| 配置 | 文言 | リンク先 |
| --- | --- | --- |
| ヘッダー右 | Contact ▸ | /contact |
| Hero | 相談する / サービスを見る | /contact, /services |
| 各サービス末尾 | この領域で相談する | /contact?topic=◯◯ |
| Casesカード末尾 | 似た課題を相談する | /contact?topic=◯◯ |
| 記事末尾 | 関連領域で相談する | /contact?topic=◯◯ |
| モバイル固定 | 無料相談 | /contact/casual |

**ポイント**: `?topic=` パラメータで、どのページから来たかをフォームに引き継ぎ、ヒアリングの精度を上げる。

---

## 5. ナビゲーション原則

1. **3クリックルール**: トップから任意のサービス詳細・任意のCTAに3クリック以内。
2. **戻り道を作る**: 詳細ページの末尾には必ず「一覧に戻る」「関連ページ」「CTA」をセットで配置。
3. **モバイルではボトムタブ風の固定CTA**: スクロール後も「無料相談」が常に視界内に。
4. **検索は当面なし**: コンテンツ量が積み上がるまではカテゴリナビで十分。

## 8. 計測（自社サイト）

| 計測対象 | 実装 |
| --- | --- |
| ページビュー | GA4 + Plausible |
| サービスカードクリック | カスタムイベント `service_card_click` |
| CTAクリック | `cta_click` + パラメータ（page, topic, position） |
| フォーム送信 | `contact_submit` |
| カジュアル面談予約 | Calendly Webhook → GA4 |
| 記事読了 | スクロール深度 75% で `article_read` |

自社サイト自体を「計測のショーケース」として作る。
